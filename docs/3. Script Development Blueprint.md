# Example use case: Minimal Violation Action script with external logging

This page demonstrates a minimal, safe Violation Action script and a repeatable workflow to validate that Execute Script works in your environment.

The goal of this example is deliberately simple.

When the DDoS engine triggers the event `DDOS_INDICATOR_EXCEED`, the script writes three structured log lines to an external syslog server.

This provides a clean foundation for more advanced scripts because you can always confirm:

- the script was invoked
- which inputs were present
- which branch was taken
- whether the script completed successfully

## Prerequisites

You need the observability pipeline to be working.

- TPS forwards syslog to the Ubuntu server
- Ubuntu receives TPS logs in `/var/log/tps.log`

Quick validation:

```bash
sudo tail -f /var/log/tps.log
```
You should see general TPS log lines. If you do not, fix syslog first before continuing.

## Minimal Building Blocks for an A10 DDoS Violation Action Script (Execute Script)

This is a didactic blueprint only (no working code yet). Use it as a consistent
structure for every script, from simple to complex. In later examples, we will
refer to these building block numbers.

**Building Block 1: Metadata and Configuration**
Purpose:
- Define what the script is and what it needs to know up front.
Typical contents
- Script name / tag for logging
- Remote syslog target (IP/port) or other observability sink
- Allowed / supported DDOS_EVENT values
- Any constants needed by the use case (prefix, thresholds, timeouts)

**Building Block 2: Input Collection**
Purpose:
- Read runtime inputs provided by the DDoS engine via environment variables.
Typical contents
- Read DDOS_* variables such as DDOS_EVENT, DDOS_DST_NAME, DDOS_PROTOCOL,
  DDOS_THRESHOLD, DDOS_ALERT_TYPE, DDOS_DST_PORT
- Apply safe defaults when a variable is missing
- Optional: normalize values (lowercasing protocol, trimming whitespace)

**Building Block 3: Trigger Check (Eligibility)**
Purpose:
- Decide whether the script should do anything for this invocation.
Typical contents
- If DDOS_EVENT is not one of the supported events, exit immediately
- Optional additional guards: protocol match, destination match, threshold match
Outcome
- No side effects when the trigger does not match (safe behavior)

**Building Block 4: Run Identifier and Start Marker (Observability Start)**
Purpose:
- Make every run traceable, even if execution is short or fails later.
Typical contents
- Create a run_id (timestamp + pid, or uuid)
- Emit a single "start" log line that includes run_id and key inputs
Note
- In many TPS environments, stdout/stderr is not captured, so logging must go to
  an external sink (e.g., syslog via UDP).

**Building Block 5: Action Execution (Business Logic)**
Purpose:
- Perform the actual use case action.
Examples
- Notify (write a structured syslog line)
- Inject/withdraw a BGP prefix
- Toggle a mitigation policy
- Call an external controller (only if outbound connectivity is supported)
Note
- Keep this block minimal at first; add one capability per iteration.

**Building Block 6: Result Reporting and Error Handling (Observability End)**
Purpose:
- Ensure the outcome is visible and actionable.
Typical contents
- Emit one "result" log line with ok=true/false
- If error: log a short reason (no stack dumps required)
- Optional: duration in milliseconds

**Building Block 7: Exit Behavior and Idempotency**
Purpose:
- Ensure repeated executions do not cause unintended side effects.
Typical contents
- For simple scripts: naturally idempotent (log-only)
- For scripts that change state: check state before change or design safe no-ops
- Always exit cleanly after reporting result


To put this into practise, here is an example Violation Action script what creates a Syslog notifivcation on a single event.

The script can also be found here: [ddos_example_min.py](../scripts/ddos_example_min.py)

Below the full script published for educational purposes.

```python
#!/usr/bin/python
"""
Example Violation Action Script (didactic): Syslog notification on a single event

This script follows the "Minimal Building Blocks" blueprint exactly.
Use case: when DDOS_EVENT == DDOS_INDICATOR_EXCEED, send one syslog line to an
external syslog server with key runtime context.

Assumptions (based on probe results in this environment)
- Python runtime: /usr/bin/python (Python 3.x)
- Third-party modules like requests are not available
- stdout/stderr may not be visible, so we use UDP syslog as output

Building Blocks referenced below:
1 Metadata and Configuration
2 Input Collection
3 Trigger Check
4 Run Identifier and Start Marker
5 Action Execution
6 Result Reporting and Error Handling
7 Exit Behavior and Idempotency
"""

# ===== Building Block 1: Metadata and Configuration =====
import os
import socket
import datetime
import time

SYSLOG_SERVER = "<<syslog-mgmt-ip>>"
SYSLOG_PORT = 514
LOG_TAG = "DDOS_EXAMPLE_MIN"

# Syslog PRI = facility(local0=16) * 8 + severity(info=6)
FACILITY_LOCAL0 = 16
SEVERITY_INFO = 6
PRI = FACILITY_LOCAL0 * 8 + SEVERITY_INFO

# Only act on this event in this simple example
ALLOWED_EVENTS = {"DDOS_INDICATOR_EXCEED"}


def _syslog_timestamp_utc():
    return datetime.datetime.utcnow().strftime("%b %d %H:%M:%S")


def _safe(value, default="-", max_len=220):
    if value is None:
        return default
    s = str(value).replace("\n", " ").replace("\r", " ")
    if len(s) > max_len:
        return s[:max_len] + "..."
    return s


def _send_syslog(message):
    """
    Sends one UDP syslog message to the external syslog server.
    """
    host = socket.gethostname()
    line = "<%d>%s %s %s: %s" % (PRI, _syslog_timestamp_utc(), host, LOG_TAG, message)
    s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
    s.sendto(line.encode("utf-8", "ignore"), (SYSLOG_SERVER, SYSLOG_PORT))
    s.close()


# ===== Building Block 2: Input Collection =====
def _collect_inputs():
    """
    Read the DDOS_* variables that are expected in this environment.
    Keep it small and predictable.
    """
    return {
        "ddos_event": os.environ.get("DDOS_EVENT", ""),
        "dst_name": os.environ.get("DDOS_DST_NAME", ""),
        "dst_port": os.environ.get("DDOS_DST_PORT", ""),
        "protocol": os.environ.get("DDOS_PROTOCOL", ""),
        "threshold": os.environ.get("DDOS_THRESHOLD", ""),
        "alert_type": os.environ.get("DDOS_ALERT_TYPE", ""),
    }


# ===== Building Block 3: Trigger Check (Eligibility) =====
def _should_act(inputs):
    """
    Only act for allowed DDOS_EVENT values.
    """
    return inputs.get("ddos_event") in ALLOWED_EVENTS


def main():
    started = time.time()

    # ===== Building Block 2: Input Collection =====
    inputs = _collect_inputs()

    # ===== Building Block 3: Trigger Check =====
    if not _should_act(inputs):
        return 0

    # ===== Building Block 4: Run Identifier and Start Marker =====
    run_id = "%d-%d" % (int(time.time()), os.getpid())
    _send_syslog(
        "stage=start run_id=%s event=%s dst=%s proto=%s threshold=%s"
        % (
            _safe(run_id),
            _safe(inputs.get("ddos_event")),
            _safe(inputs.get("dst_name")),
            _safe(inputs.get("protocol")),
            _safe(inputs.get("threshold")),
        )
    )

    ok = True
    err = ""

    try:
        # ===== Building Block 5: Action Execution =====
        # Simple action: emit a single structured notification line.
        _send_syslog(
            "stage=action run_id=%s action=notify event=%s dst=%s dst_port=%s proto=%s threshold=%s alert=%s"
            % (
                _safe(run_id),
                _safe(inputs.get("ddos_event")),
                _safe(inputs.get("dst_name")),
                _safe(inputs.get("dst_port")),
                _safe(inputs.get("protocol")),
                _safe(inputs.get("threshold")),
                _safe(inputs.get("alert_type")),
            )
        )

    except Exception as e:
        # ===== Building Block 6: Result Reporting and Error Handling =====
        ok = False
        err = _safe(e)

    # ===== Building Block 6: Result Reporting and Error Handling =====
    duration_ms = int((time.time() - started) * 1000)
    _send_syslog(
        "stage=end run_id=%s ok=%s duration_ms=%s err=%s"
        % (_safe(run_id), str(ok).lower(), str(duration_ms), _safe(err))
    )

    # ===== Building Block 7: Exit Behavior and Idempotency =====
    # This script is idempotent because it only logs.
    return 0 if ok else 1


if __name__ == "__main__":
    raise SystemExit(main())
```

## Script behavior

This example script uses the same building blocks described earlier.

- It reads DDOS_* environment variables
- It only acts if DDOS_EVENT == DDOS_INDICATOR_EXCEED
- It sends output to syslog via UDP
- It logs three phases using a shared run_id

Log phases:
- `stage=start` confirms invocation and captures key inputs
- `stage=action` confirms the script took the expected branch and shows the planned action
- `stage=end` confirms completion and reports ok true false and duration

## Step-by-step usage
**Step 1: Upload the script in A10 Control**

At A10 Control, create a new Violation Action Script object.

- Type: Python
- File: `ddos_example_min.py` (the example script)

Save and push the configuration so the script is imported on the TPS.

**Step 2: Create a Violation Action referencing the script**

Create a new Violation Action.

- Action type: Execute Script

- Script: `ddos_example_min.py`

**Step 3: Attach the Violation Action to a reproducible trigger**

Attach the Violation Action to a threshold or indicator that you can trigger repeatedly.

This example expects to see:

 - `DDOS_EVENT = DDOS_INDICATOR_EXCEED`

If you are unsure which events you get, run the Python probe first and confirm the event name.

**Step 4: Trigger the test**

Trigger your ICMP indicator exceed scenario.

**Step 5: Verify output on Ubuntu**

On the Ubuntu syslog server, watch for the script output.
```bash
sudo tail -f /var/log/tps.log | grep DDOS_EXAMPLE_MIN
```
### Expected result

For each matching event, you should see exactly three log lines with the same run_id.

The following is a real example output from the test run.
```Plain text
Mar  5 09:22:01 10.25.12.132 GJW-MITIGATOR DDOS_EXAMPLE_MIN: stage=start run_id=1772702500-19749 event=DDOS_INDICATOR_EXCEED dst=zone1 proto=icmp threshold=30
Mar  5 09:22:01 10.25.12.132 GJW-MITIGATOR DDOS_EXAMPLE_MIN: stage=action run_id=1772702500-19749 action=notify event=DDOS_INDICATOR_EXCEED dst=zone1 dst_port=1 proto=icmp threshold=30 alert=FAIL
Mar  5 09:22:01 10.25.12.132 GJW-MITIGATOR DDOS_EXAMPLE_MIN: stage=end run_id=1772702500-19749 ok=true duration_ms=0 err=
```
Interpretation:

- The script was invoked and matched the trigger event
- The runtime inputs were captured correctly
- The action phase executed
- The script completed successfully with ok true

### Notes and troubleshooting

If you see no `DDOS_EXAMPLE_MIN` lines:

- Verify the event is actually DDOS_INDICATOR_EXCEED using the Python probe

- Confirm the Violation Action is attached to the correct threshold

- Confirm the script was imported on TPS and executed (TPS show log should show Script Running and Success)

If you see only stage start but not stage end:

- The script may be failing mid execution

- Check the TPS log stream around the time of the start line
